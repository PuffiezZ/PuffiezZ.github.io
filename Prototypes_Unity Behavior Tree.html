This project is a final assignment that uses a behaviour tree tool. I decided to use the built-in Unity 6 Behaviour Tree Package as my main tool. The core idea of this design is to combine an FSM to handle the complexity of behaviours, while using the Behaviour Tree to decide which state to execute based on external data.
BT + FSM
Behavior Tree is a graph structure based on a root-tree algorithm. It allows for faster, more efficient execution and provides an easier-to-understand structure without relying on hard-coded logic. The Finite State Machine can then handle only the complex behaviours within each specific state.

This thing calls: Hybrid Approach
Default State
Both character is using the same enum state [Random Patrol, Chase, Attack, Strong Attack, Ultimate]. They are doing normal attack and Strong Attack by cooldown time!
Samurai Behavior
He is the main AI leader. He reads the Minotaur AI status from shared public variables through the Blackboard system. He also has an internal cooldown for his ultimate ability. Once the cooldown is fully charged, he triggers the Minotaur to perform a pre-ultimate pose before releasing his ultimate attack.
Minitour Behavior
He is the secondary AI character. He performs a charging attack and a fire-breath attack as his powerful abilities. He receives the ultimate signal from the Samurai AI, and once triggered, he unleashes his own ultimate attack.
